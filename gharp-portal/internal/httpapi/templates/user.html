🪉 Connect a repository

  1. Create your GitHub App. Opens GitHub and registers an App scoped to your slot (its webhook points at your runner). gharp drives the flow — just confirm on GitHub.
    Open setup ↗
  2. Install the App on the repository (or whole org) you want runners for — gharp shows an “Install on GitHub” link after creation.
  3. Point your workflow at the runners — in .github/workflows/<name>.yml: 
    jobs:
  build:
    runs-on: [self-hosted]
    steps:
      - uses: actions/checkout@v4
      - run: echo "hi from my gharp runner"
  4. Push or trigger the workflow. The job shows up on this board (Queued → Running → Done); a fresh ephemeral runner spins up for each job.
Private repositories only — public repos are dropped before dispatch (fork-PR RCE guard).
